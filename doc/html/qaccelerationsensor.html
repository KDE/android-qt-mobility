Detailed Description

The QAccelerationSensor class reports on linear acceleration along the X, Y and Z axes.

The acceleration sensor returns acceleration values along 3 axes. The scale of the values is milli-Gs. The axes are arranged as follows.

             +z
              |
              |      +y
              |     /
              |----/----
             /| NOKIA  /|
            //|--/--- / |
           // | /   //  /
          //  |/   //  /
         //   '--------------- +x
        //       //  /
       //       //  /
      /---------/  /
     /    O    /  /
    /         /  /
    ----------  /
    |_________!/

A monoblock device sitting at rest, face up on a desk will experience the force of gravity as approximately -1000 on the Z axis.

See also QAccelerationReading.

Construct a sensor instance with specified parent. If the id is passed the sensor will connect to that specific sensor, otherwise the default will be used.

QAccelerationSensor::~QAccelerationSensor ()   [virtual]

Destroy the sensor. Stops the sensor if it has not already been stopped.

void QAccelerationSensor::accelerationChanged ( const QAccelerationReading & reading )   [signal]

This signal is emitted when a new acceleration reading comes in.

Note that this signal should not be used if you are requesting high-frequency updates as signal delivery is quite slow.

void QAccelerationSensor::addListener ( QAccelerationListener * listener )

Add a listener to the sensor. The listener will be invoked every time a new reading is available.

Note that the sensor does not take ownership of the listener. It is the caller's responsibility to ensure the listener remains valid until the sensor is destroyed or the listener is removed via QAccelerationSensor::removeListener().
