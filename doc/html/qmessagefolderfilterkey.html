Detailed Description

The QMessageFolderFilterKey class defines the parameters used for querying a subset of all available folders from the message store.

A QMessageFolderFilterKey is composed of a folder property, an optional comparison operator and a comparison value. The QMessageFolderFilterKey class is used in conjunction with the QMessageStore::queryFolders() and QMessageStore::countFolders() functions to filter results which meet the criteria defined by the key.

QMessageFolderFilterKeys can be combined using the logical operators (&), (|) and (~) to create more refined queries.

See also QMessageStore and QMessageFolder.

bool QMessageFolderFilterKey::isEmpty () const

Returns true if the key remains empty after default construction; otherwise returns false.

An empty key matches all folders.

The result of combining an empty key with a non-empty key is the original non-empty key. This is true regardless of whether the combination is formed by an AND or an OR operation.

The result of combining two empty keys is an empty key.
