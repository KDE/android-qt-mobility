Detailed Description

The class QGeoNetworkManager is a subclass of QGeoEngine for use with Nokia maps.

This class extends QGeoEngine. It handles all communication with the REST servers. Internally it uses QNetworkAccessManager for network communication, and it handles all mapping between raw network requests/replies and geo requests/replies.

QMapTileReply * QGeoNetworkManager::get ( const QMapTileRequest & request )   [virtual]

Reimplemented from QGeoEngine::get().

Submits a map tile request request and returns the corresponding QMapTileReply.

Note: Due to the potentially asynchronous nature of requests, you should never directly start working with the returned reply, but instead wait for the finished(QMapTileReply*) signal. Note: If the request can be served from the tile cache, this method will directly emit finished(QRouteReply*). If you need the returned QMapTileReply object first, you should connect to finished(QRouteReply*) with Qt::QueuedConnection.
