Detailed Description

The QSensor class represents a single hardware sensor.

The life cycle of a sensor is typically:

  • Create a sub-class of QSensor on the stack or heap.
  • Setup as required by the application.
  • Start receiving values.
  • Sensor data is used by the application.
  • Stop receiving values.

Please see the individual sensor classes for details on their use.

Member Type Documentation

enum QSensor::UpdatePolicy
flags QSensor::UpdatePolicies

This enum is used to indicate to the sensor how often data will be collected. Note that most sensors will only support one sensitivity. Setting an update policy that the sensor does not support will result in undefined behaviour. You can determine the policies the sensor supports with the QSensor::supportedPolicies() method.

ConstantValueDescription
QSensor::Undefined0x00The sensor has no specific update policy.
QSensor::OccasionalUpdates0x01Updates are delivered only occasionally.
QSensor::InfrequentUpdates0x02Updates are delivered every now and then.
QSensor::FrequentUpdates0x04Updates are delivered frequently.
QSensor::TimedUpdates0x08Updates are delivered at the specified time interval.
QSensor::PolledUpdates0x10Updates are retrieved when the device is polled.

The UpdatePolicies type is a typedef for QFlags<UpdatePolicy>. It stores an OR combination of UpdatePolicy values.

void QSensor::setUpdatePolicy ( UpdatePolicy policy, int interval = 0 )

Change the update policy of the sensor. If using TimedUpdates you must also supply the interval. Note that not all sensors support changing the update policy. If you set a policy that the sensor does not support the behaviour is undefined.

See also updatePolicy() and supportedPolicies().

bool QSensor::start ()   [slot]

Start retrieving values from the sensor.

Note that some sensors require exclusive access so this function may fail and return false.

Also note that some sensors may not honour settings set after this method is called.
