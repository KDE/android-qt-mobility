Telephony API

The Qt Telephony API enables access to the telephony event services to provide telephony call notification that other applications can use to be notified. Using this API you can further get information abaout the status of all existing calls.

Overview

The telephony library provides a set of function for accessing existing calls and their information like. Using this API you are able to queries the current telephony state. e.g. no call, ringing, call in-progress and call on-hold. Signals are provided to inform other application of changes of a call status. e.g. ringing, on-hold, terminating or of a change of the list of the current existing calls. e.g. call is ending. The Application can query call information using this API. e.g. call type, phone number, SIP id, link to the caller contact.

Information and Events of Calls

The library provides a number of classes and signals to assist with the access of the call data. Furthermore the library provides a couple of signals to provide notification of call changes.

Classes and to assist sending call status change notification and reading the call information:

ClassDescription
QTelephonyEventRepresents the call list container and the notifier about call changes.
QCallInfoRepresents a call including the type, status and other information of the call.

Call Status

The library provides an enumerator to define the status of a call.

Enum values relating to call status:

enumvalueDescription
CallStatus::UnknownStatusThe status of the call is unknown.
CallStatus::NoCallNo active calls are available.
CallStatus::RingingCall is ringing.
CallStatus::InProgressCall is in progress (dialing).
CallStatus::OnHoldCall is on hold.

Call Type

The library provides an enumerator to differ between types of calls.

Enum values relating to call types:

enumvalueDescription
CallType::UnknownTypeThe type of the call is unknown.
CallType::VOIPThe call is a VOIP call.
CallType::VoiceThe call is a voice call.
CallType::VideoThe call is a video call.
