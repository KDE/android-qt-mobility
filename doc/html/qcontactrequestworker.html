Detailed Description

The QContactRequestWorker class provides a common thread worker queue for QContact asynchronous requests.

A QContactRequestWorker consists of a QContactAbstractRequest request queue.

An instance of the QContactRequestWorker class is a thread object which is dedicated for processing asynchronous contact requests. All of these requests are instances of derived classes from QContactAbstractRequest. These request instances can be added to, removed/cancelled from the worker thread's internal request queue. Once these requests are processed by the worker thread, the related signals will be emitted and all these requests' waiting clients will also wake up.

See also QContactAbstractRequest.

void QContactRequestWorker::run ()   [virtual]

Reimplemented from QThread::run().

Worker thread's main working loop. Runs forever until the stop() function called. The worker thread initionally waits on the empty request queue, once the request queue is not empty, takes the first element of the queue, then calls related request processing functions based on the request type. Once the request was processed, update the request's status and wakes up any threads which waiting for this request.

See also stop(), processContactFetchRequest(), processContactLocalIdFetchRequest(), processContactSaveRequest(), processContactRemoveRequest(), processContactRelationshipFetchRequest(), processContactRelationshipSaveRequest(), processContactRelationshipRemoveRequest(), processContactDetailDefinitionFetchRequest(), processContactDetailDefinitionSaveRequest(), and processContactDetailDefinitionRemoveRequest().

void QContactRequestWorker::stop ()

Stops the worker thread. The worker thread may still run a while until it tries to process next request. If the worker thread was waiting on an empty queue, wakes it up.

See also run().

bool QContactRequestWorker::waitRequest ( QContactAbstractRequest * req, int msecs )

Blocks the caller until the given request req has been completed by the worker thread or worker thread signals that more partial results are available for the request, or until msecs milliseconds has elapsed. If msecs is zero, this function will block indefinitely. Returns true if the request was cancelled or completed successfully within the given period, otherwise false.

See also QContactAbstractRequest::waitForFinished() and QContactAbstractRequest::waitForProgress().
